End-to-End Agent Flow

The agent follows the same sequence each cycle. The dashboard runs the workflow in-process, and the command line can run the same flow with npm run workflow:once or npm run workflow:loop.

1

Discover

SAP registry discovery is queried for AgenticBro-compatible tools. The workflow also confirms the agent is registered and active on SAP mainnet.

2

Select

A rotating watchlist selects one Solana token or project target per cycle, such as USDC, SOL, POPCAT, BONK, Orca, or Raydium.

3

Verify

The workflow checks Synapse Sentinel context and confirms an active Sentinel escrow on SAP, proving Sentinel is part of the execution path.

4

Analyze

Ace Data Cloud AI text generation produces a structured risk assessment with risk level, confidence, flags, summary, and recommendation.

5

Settle

The agent settles successful calls from a self-funded SAP x402 escrow, creating visible payment volume on SAP mainnet.

Services Used and Why

The category rewards general SAP payment volume, so the services are chosen to make each paid call meaningful: verify network trust, analyze the target, then settle calls through escrow.

SAP Registry

Tool discovery and agent identity

AgenticBro fetches SAP registration state, active capabilities, protocols, and pricing. This proves the agent exists on SAP mainnet before work begins.

Synapse Sentinel

Network trust verification

The workflow attempts a Sentinel service call and records a registry/x402 fallback when the public HTTP endpoint is unavailable. Sentinel is active and escrow-backed.

Ace Data Cloud AI

Risk analysis capability

The AI text generation endpoint returns JSON risk assessments for selected tokens and projects. This is the agent's core intelligence step.

How Payments Work

Payment volume is not simulated. The agent creates an SAP x402 escrow, executes a service cycle, then settles calls on-chain as the registered agent owner.

Escrow

AgenticBro uses a self-funded escrow for its registered SAP agent. The escrow starts with 5000000 lamports and is priced at 1000 lamports per call.

Why It Is Autonomous

After configuration and funding, the agent does not need manual selection, manual analysis, or manual settlement. One loop performs the full lifecycle.

No Manual Target Picking

The workflow rotates through a watchlist and chooses the next token or project automatically each cycle.

No Manual API Calls

Discovery, Sentinel verification, AI analysis, and payment headers are generated by the runtime, not by a human operator.

No Manual Settlement

Successful service calls are counted and settled through SAP x402 during the cycle, with timeout recovery for the Synapse RPC confirmation path.
